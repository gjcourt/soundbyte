# reference/

Information you look things up in — wire-protocol specs, auth flows, configuration tables.

**Put here:**
- UDP wire-format reference (chunk shape, frame size, ordering semantics).
- Auth / HMAC token semantics and lifecycle.
- Configuration env-var tables.

**Do not put here:**
- Runbook steps — `operations/`.
- Architecture overview — `architecture/`.
- Spike output — `research/`.

**Naming convention:** `<yyyy-mm-dd>-<topic>.md`
Examples: `2026-05-02-api.md`, `2026-05-02-authentication.md`.

**Allowed `status:` values:** `Stable`, `Superseded`.

Date prefix is bumped when the doc is materially revised.
